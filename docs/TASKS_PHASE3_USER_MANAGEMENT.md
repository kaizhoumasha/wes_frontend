## 2. 开发边界

### 2.1 本轮必须完成（P0）

- `/admin/users` 页面接入
- 用户列表分页查询
- 用户名/邮箱基础筛选
- 新建用户
- 编辑用户
- 删除用户（软删除）
- 页面权限与按钮权限控制
- 表单校验与提交流程闭环

### 2.2 本轮可选增强（P1）

- 用户详情抽屉
- 回收站页面/标签页
- 恢复用户
- 缓存统计信息展示

### 2.3 本轮不做

- 角色分配编辑
- 批量导入导出
- 高级筛选 DSL UI
- 用户模块 Pinia Store
- 动态路由系统重构

## 9. 开发建议

### 9.1 推荐分支提交节奏

建议按功能块逐步提交：

1. `feat(user): add admin users route and page shell`
2. `feat(user): implement users list and filters`
3. `feat(user): implement create and edit dialog`
4. `feat(user): implement delete flow and permission gating`
5. `refactor(user): polish interactions and empty states`

### 9.2 每阶段完成后建议验证

- 路由接入后：验证页面可进入
- 列表完成后：验证查询与分页
- 表单完成后：验证创建/编辑
- 删除完成后：验证状态回收
- 全部完成后：执行类型检查与 ESLint
