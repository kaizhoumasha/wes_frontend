# 通用 CRUD 页面设计规范

> **版本**: v1.3 (定稿)  
> **来源**: `docs/PHASE3_USER_MANAGEMENT_GUIDE.md` 第17章  
> **适用范围**: 项目中所有标准的"列表-增删改查"页面

---

## 核心布局原则

### 上-中-下三段式结构

页面采用经典的**上-中-下**三段式布局，结合**垂直弹性**设计：

1. **顶部 (Top)**: 集成工具栏
   - 页面大脑，承载标题、操作、搜索、视图控制
2. **中部 (Middle)**: 数据表格区
   - 核心内容展示区，**弹性高度**（占据剩余空间）
3. **底部 (Bottom)**: 分页控制区
   - 固定高度，右对齐

### 垂直高度响应式原则

1. **固定区域**: 工具栏 + 分页栏（内容决定高度）
2. **弹性区域**: 表格区（`flex-grow: 1`，自动占据剩余空间）
3. **内部滚动**: 表头固定，表体内部滚动（非整页滚动）

## 集成工具栏规范 (4段式分区)

从左到右分为四个功能区：

### 1. 标题区

- **构成**: `[页面图标] 主标题` (例: `[i] 用户管理`)
- **职责**: 页面身份标识和上下文

### 2. 数据操作区

- **构成**: `1~2个主要操作按钮 + [...] 更多操作菜单`
- **规范**:
  - 最高频操作（如"新增"）始终可见
  - 次要操作（批量删除、导入、导出）收纳在"更多"菜单

### 3. 智能搜索区

- **构成**: `SmartSearchBar` 组件
- **强制**: 所有列表页**必须复用**此组件
- **禁止**: 手写独立的搜索输入框

### 4. 表格控制区

- **构成**: 一组图标按钮，居右对齐
- **功能**: `[⟳] 刷新` `[⛶] 全屏` `[↕] 密度` `[⚙] 列配置`
- **强制**: 每个图标按钮**必须**提供详细的 Tooltip 提示

## 响应式布局规范

### 断点策略：逐级堆叠 + 折叠优化

#### 大屏幕 (Large, >1280px)

```
| [i] 标题 | [+ 新增] [...] | [ 搜索... ] | [⟳][⛶][↕][⚙] |
```

所有功能区在一行完整显示

#### 中屏幕 (Medium, <1280px)

```
| [i] 标题                                                     |
|--------------------------------------------------------------|
| [+ 新增] [...] | [ 搜索... ]                   | [...]        |
```

- 标题区换行（独立一行）
- 表格控制区合并为 `[...]` 菜单

#### 小屏幕 (Small, <768px)

```
| [i] 标题                                                     |
|--------------------------------------------------------------|
| [ 搜索...                                     ]              |
|--------------------------------------------------------------|
| [+ 新增] [...]                                   | [...]      |
```

- 智能搜索区换行（独立一行）

#### 极小屏幕 (X-Small, <480px) - 移动端

```
| [i] 标题                                        | [🔍] [...] |
|--------------------------------------------------------------|
| (表格内容...)                                                |
|                                                     (+) (FAB)|
```

- 智能搜索区折叠为 `[🔍]` 图标
- 数据操作 + 表格控制合并到 `[...]` 菜单
- `+ 新增` 变为右下角浮动操作按钮 (FAB)

## 关键交互细节

### 1. 表格中间态

**加载中 (Loading)**

- **强制**: 使用骨架屏 (Skeleton Screen)
- **禁止**: 使用简单的 loading 文本或转圈动画

**空状态 (Empty)**

- **强制**: 显示独立空状态组件
- **包含**: 图标 + 文本提示 + 可选操作指引

**错误 (Error)**

- **强制**: 显示错误提示组件 + "重新加载" 操作

### 2. 批量操作

**触发机制**

- 表格第一列提供复选框
- 至少选中一行时，批量操作才被激活

**状态反馈**

- 工具栏中的 `[- 批量删除]` 等按钮变为可用
- **标题区**变为**上下文操作区**，显示：
  - "已选中 N 项"
  - 提供"取消选择"快捷操作

**示例 (选中行后)**

```
| 已选中 5 项 [取消选择] | [- 批量删除] [...] | (搜索区和控制区可被临时隐藏) |
```

### 3. 新建/编辑流程

- **推荐**: 使用模态对话框 (`ElDialog`) 或抽屉 (`ElDrawer`)
- **禁止**: 整页跳转（破坏操作沉浸感）

## 实施检查清单

开发新 CRUD 页面时，必须确认：

- [ ] 采用上-中-下三段式布局
- [ ] 表格区域使用弹性高度（`flex-grow: 1`）
- [ ] 表格内部滚动，表头固定
- [ ] 工具栏包含 4 个分区（标题/操作/搜索/控制）
- [ ] 使用 `SmartSearchBar` 组件（不手写搜索）
- [ ] 所有图标按钮提供 Tooltip
- [ ] 实现骨架屏加载态
- [ ] 实现空状态和错误状态组件
- [ ] 支持批量操作和上下文操作区
- [ ] 响应式布局测试（>1280px / <1280px / <768px / <480px）
